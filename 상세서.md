# 사주 명리학 AI 리포트 자동화 시스템 UI/UX 상세 명세

## 1. 제품 UX 방향

| 항목 | 값(HEX·px·ms) | 사용처 | 규칙 |
| --- | --- | --- | --- |
| 제품 성격 | 내부 운영툴 / PC Web 전용 | 전체 서비스 | 외부 홍보성 연출보다 입력 정확도, 생성 이력 추적, PDF 발행 안정성을 우선한다. |
| UX 목표 | 입력 → 검증 → 생성 → 편집 → 발행 5단계 | 전체 플로우 | 운영자가 한 케이스를 처음부터 최종 PDF까지 한 화면 체계 안에서 완료하도록 설계한다. |
| 시각 톤 | `Deep Indigo + Warm Gold + Paper White` | 브랜드 인상 | 명리학의 전통성은 유지하되, 분석 도구처럼 정돈된 밀도와 신뢰감을 유지한다. |
| 조작 원칙 | 3클릭 이내 / 2스크롤 이내 | 대시보드, 케이스 상세 | 자주 쓰는 기능은 사이드바 1단계, 탭 2단계, 액션 버튼 3단계 이내에 둔다. |
| 정보 위계 | 1차: 상태 / 2차: 본문 / 3차: 보조메타 | 카드, 테이블, 에디터 | 상태값과 다음 액션을 최상단에 고정하고, 설명성 문구는 보조 위치로 내린다. |
| 오류 복구 | 인라인 오류 + 상단 배너 + 로그 기록 | 폼, 생성 작업, PDF 발행 | 오류는 발생 위치 1차 노출, 페이지 상단 요약 2차 노출, 작업 로그 3차 기록으로 중복 확인한다. |
| 데이터 신뢰 표시 | 상태 배지 24px / 로그 타임라인 14px | API 연동, 정규화, LLM 생성 | 모든 자동 단계는 `대기/진행중/완료/검토필요/실패` 5상태 중 하나를 반드시 가진다. |
| 문서 편집 원칙 | 3패널 편집 / 자동저장 15000ms | 초안 편집 워크스페이스 | 좌측 구조, 중앙 본문, 우측 속성 패널을 분리하고 저장 여부를 항상 가시화한다. |
| PDF 발행 원칙 | 미리보기-템플릿-발행로그 동시 표시 | PDF 퍼블리싱 화면 | 발행 전 미리보기, 템플릿 선택, 출력 옵션을 한 화면에서 비교 가능해야 한다. |

## 2. 정보 구조 및 화면 구성

| 항목 | 값(HEX·px·ms) | 사용처 | 규칙 |
| --- | --- | --- | --- |
| 로그인 | 우측 로그인 카드 `420px`, 좌측 소개 패널 `1fr` | 인증 진입 화면 | 폐쇄형 시스템임을 명확히 표시하고, 좌측에는 시스템 공지·최근 발행 상태만 노출한다. |
| 대시보드 | KPI 카드 4열 / 작업목록 8:4 분할 | 첫 화면 | 오늘 생성 건수, 검토 필요 건수, 실패 건수, 발행 완료 건수를 1행에 고정한다. |
| 고객 등록 | 12컬럼 폼 / 섹션 카드 단위 | 고객 기본정보 입력 | `기본정보 → 출생정보 → 옵션 → 메모` 순서로 진행하고 섹션 헤더에 필수 여부를 표시한다. |
| 케이스 상세 | 상단 요약 바 `160px`, 하단 탭 콘텐츠 | 개별 고객 작업 | 고객 식별 정보, 생성 상태, 마지막 수정자, 최종 PDF 버전을 상단 요약 바에 고정한다. |
| AI 초안 생성 | 좌측 파라미터 / 중앙 프롬프트 / 우측 결과로그 | LLM 생성 관리 | 프롬프트 버전, 모델, 분량(30/100p), 생성 범위를 동시에 확인하고 재실행 이력을 남긴다. |
| 웹 에디터 | 좌측 목차 `280px`, 중앙 에디터 `min 880px`, 우측 검사기 `360px` | 초안 수정 | 장별 이동, 본문 편집, 메타정보 편집을 병렬로 수행하되 본문 영역의 시선을 방해하지 않는다. |
| PDF 퍼블리싱 | 좌측 페이지 썸네일 `120px`, 중앙 미리보기 `min 820px`, 우측 발행옵션 `320px` | 최종 출력 | 미리보기는 A4 비율을 유지하고, 발행 옵션 변경 시 1000ms 이내 부분 재렌더링한다. |
| 이력 관리 | 검색 바 `320px`, 필터 3~5개, 테이블 | 관리자 전용 대시보드 | 고객명, 생성일, 템플릿, 발행상태, 다운로드 액션을 1행에서 처리 가능해야 한다. |
| 설정/권한 | 2열 카드 레이아웃 `7:5` | 관리자 설정 | API 키 노출 금지, 역할별 권한 범위, 프롬프트 버전 관리, 템플릿 관리 탭으로 분리한다. |

| 키보드 탐색 | `Tab / Shift+Tab / Enter / Esc` | 앱 전역 | 모달, 드로어, 메뉴, 테이블 셀 이동까지 모두 키보드로 수행 가능해야 한다. |
| 라벨 연결 | `for/id` 명시 | Input, Textarea, Select | placeholder를 라벨 대체로 사용하지 않는다. |
| 오류 전달 | 색상 + 아이콘 + 문장 | 폼, 생성, 발행 | 오류 상태는 색상만으로 전달하지 않는다. |
| 상태 알림 | `aria-live="polite"` | Toast, 자동저장 상태 | 저장 완료/실패, 생성 완료/실패는 스크린 리더에 읽혀야 한다. |
| 모달 포커스 트랩 | 필수 | Modal, Drawer | 열린 동안 배경 요소에 포커스가 가면 안 된다. |
| 표 구조 | `thead / tbody / scope` | Table | 정렬 가능한 컬럼은 현재 정렬 방향을 보조기기에 전달한다. |
| 입력 도움말 | `aria-describedby` | 헬퍼/오류 텍스트 | 필드와 설명문은 항상 연결한다. |
| 클릭 타깃 | `44px 이상` | 버튼, 체크박스, 아이콘 | 아이콘 단독 버튼도 44px 미만으로 만들지 않는다. |
| 모션 감소 | `prefers-reduced-motion` 대응 | 전환, skeleton | 큰 이동 애니메이션과 패럴랙스 효과를 금지한다. |
| 언어 설정 | `lang="ko"` | 문서 루트 | 리포트 편집 영역이 다른 언어를 포함하면 구간별 lang 속성을 부여한다. |

| 기본 글꼴 | `Pretendard Variable` 사용 가능 | UI 전역 | 내부망 환경에도 설치 또는 웹폰트 제공이 가능하다고 가정한다. |
| 인증 체계 | 사내 계정 기반 로그인 | 로그인/권한 | 외부 소셜 로그인이나 고객 셀프 접근은 고려하지 않는다. |
